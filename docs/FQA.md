## FAQ 常見問題

### Q1: 如何選擇使用 UART 還是 I2C？

**A**: 選擇建議：

**使用 UART/USB 當**：

- 連接到 PC 進行調試和測試
- 需要人機互動操作
- 開發階段需要查看即時數據
- 使用終端程式控制模組

**使用 I2C Slave 當**：

- 嵌入式系統整合（MCU 對 MCU）
- 需要快速讀取二進位數據
- 機器人、AGV 等自動化應用
- 節省通訊線材（僅需 2 條線）
- 需要多個感測器掛載同一 I2C 匯流排

### Q2: I2C 模式下如何切換回 UART 模式？

**A**: 需要重新上電或硬體重置。I2C 模式一旦啟用會保持到下次重置。這是為了避免介面衝突和確保穩定運行。

### Q3: I2C 通訊失敗怎麼辦？

**A**: 故障排除步驟：

1. **檢查硬體連接**：

   - 確認 SDA (PA0) 和 SCL (PA1) 連接正確
   - 檢查是否有 4.7kΩ 上拉電阻到 3.3V
   - 確認 GND 共地連接
2. **檢查 I2C 設定**：

   - 確認 I2C 地址為 0x52 (7-bit)
   - 確認時鐘速率設為 200kHz 或更低
   - 檢查是否正確使用 7-bit 地址模式
3. **檢查線材品質**：

   - 使用短線材（建議 < 30cm）
   - 避免平行長距離佈線
   - 遠離高頻干擾源
4. **程式碼檢查**：

   - 確認使用正確的讀寫函式
   - 檢查超時設定是否充足
   - 驗證資料型態和位元序（Little-Endian）

### Q4: UPDATE_SENSORS 位元無法清除？

**A**: 可能原因和解決方法：

1. **等待時間不足**：增加超時時間至 800ms 以上
2. **感測器初始化失敗**：重新上電初始化模組
3. **I2C 通訊錯誤**：檢查硬體連接和上拉電阻
4. **讀取時序問題**：在讀取前加入適當延遲（10ms）

### Q5: 羅盤校正數據寫入後沒有生效？

**A**: 正確步驟：

1. 先寫入 compassMinX/MaxX/MinY/MaxY/MinZ/MaxZ 到對應寄存器（0x29-0x33）
2. 讀取 controlBits (0x01)
3. 設定 bit 4 (SET_CMPS_CAL_DATA)
4. 等待 bit 4 自動清除（表示完成）
5. 校正數據會自動保存到 EEPROM，重新上電後保留

參考 test_hub5168_i2c.ino 中的 `ca` 命令實作。

### Q6: 可以同時使用 UART 和 I2C 嗎？

**A**: 不行（DEBUG 模式除外）。當 I2C 通訊發生時，UART 會自動停用以：

- 節省系統資源
- 避免介面衝突
- 確保即時性和穩定性

如需同時使用，需要在韌體中啟用 DEBUG 模式並重新編譯。

### Q7: 如何讀取羅盤校正數據？

**A**:

```cpp
// 讀取校正數據（6個 int16_t 值）
int16_t minX = (int16_t)i2c_read_word(0x52, 0x29);
int16_t maxX = (int16_t)i2c_read_word(0x52, 0x2B);
int16_t minY = (int16_t)i2c_read_word(0x52, 0x2D);
int16_t maxY = (int16_t)i2c_read_word(0x52, 0x2F);
int16_t minZ = (int16_t)i2c_read_word(0x52, 0x31);
int16_t maxZ = (int16_t)i2c_read_word(0x52, 0x33);
```

參考 test_hub5168_i2c.ino 中的 `cd` 命令實作。

### Q8: 模組無法連線？

**A**: 請檢查：

1. USB 線是否正常連接
2. 驅動程式是否已安裝
3. COM 埠號是否正確
4. 串口是否被其他程式佔用
5. 波特率是否設定為 115200

### Q9: 數據輸出不穩定？

**A**: 可能原因：

1. 感測高度設定不正確，使用 `ATH+` 調整
2. 表面材質不適合光流感測（建議使用非反光、有紋理的表面）
3. 移動速度過快
4. 環境光線變化過大

### Q10: 羅盤讀數不準確？

**A**: 請執行羅盤校正：

1. 遠離強磁場干擾源（馬達、喇叭等）
2. 執行 `AT0C+8` 進行校正
3. 校正時緩慢旋轉模組，確保覆蓋所有方向
4. 校正完成後重新測試

### Q11: 座標漂移問題？

**A**: 解決方法：

1. 定期執行 `AT0G+` 重置零點
2. 確認感測高度設定正確
3. 確保表面平整且適合光流感測
4. 使用羅盤校正功能降低角度漂移

### Q12: 如何提高位移精度？

**A**: 建議：

1. 正確設定感測高度（`ATH+` 指令）
2. 使用適當的表面材質（非反光、有紋理）
3. 保持穩定的照明環境
4. 定期校正電子羅盤
5. 避免快速移動或劇烈震動

### Q13: USART1 無法使用？

**A**:

1. 確認 RX (PA3) 腳位已上拉至高電平（3.3V 或 5V）
2. 檢查 TX/RX 是否正確連接
3. 確認外部設備的波特率為 115200

### Q14: 如何儲存校正數據？

**A**:

- 羅盤校正數據會自動儲存至 EEPROM
- 重新上電後會自動載入
- 如需手動設定，可使用 `ATI&` 指令

### Q15: JSON 輸出格式解析錯誤？

**A**:

1. 確認已使用 `ATJ+` 啟用 JSON 格式
2. 檢查串口接收緩衝區大小是否足夠
3. 確認沒有遺漏換行符號
4. 使用標準 JSON 解析庫

### Q16: 如何重置為出廠設定？

**A**:

1. 重新燒錄韌體（會清除 EEPROM）
2. 或手動重置：
   - 執行 `AT0G+` 重置感測器
   - 執行 `AT0C+8` 重新校正羅盤
   - 執行 `ATH+10` 設定預設高度
   - 執行 `ATRPT+200` 設定預設報告間隔
